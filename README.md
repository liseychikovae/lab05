# lab05

Лабораторная работа Лисейчиковой Екатерины

## Отчет по лабораторной работе V

### Тема

Изучение фреймворков для тестирования на примере GTest.

### Цель работы

Цель работы - добавить модульные тесты и подключить GoogleTest к существующему CMake-проекту.

### Используемое окружение

- основной целевой сценарий: Linux;
- язык: C++11;
- система сборки: CMake;
- дополнительные инструменты добавляются по теме конкретной лабораторной;
- персональные токены и временные адреса не сохраняются в репозитории.

### Структура проекта

- `formatter_lib` - базовая статическая библиотека форматирования;
- `formatter_ex_lib` - расширенная библиотека, использующая `formatter`;
- `solver_lib` - библиотека для решения квадратного уравнения;
- `hello_world_application` - демонстрационное приложение;
- `solver_application` - приложение для вывода корней уравнения;
- `tests` - модульные тесты, начиная с lab05;
- `docs` и `scripts` - дополнительные материалы поздних лабораторных.

### Ход выполнения

- добавлена опция `BUILD_TESTS`; 
- создан каталог `tests` с отдельным `CMakeLists.txt`; 
- GoogleTest подключается через CMake `FetchContent`; 
- написаны тесты для `formatter_ex` и `solver`; 
- CI из lab04 расширен запуском `ctest`.

### Команды для проверки

```bash
cmake -S . -B _build -DCMAKE_BUILD_TYPE=Release -DBUILD_TESTS=ON
cmake --build _build
ctest --test-dir _build --output-on-failure
./_build/tests/check
```

### Домашнее задание

Домашнее задание выполнено внутри этого же репозитория. Подробное описание вынесено в `HOMEWORK.md`.

Кратко по выполнению:

- тест `FormatterEx.WrapsText` проверяет, что расширенный formatter сохраняет исходный текст;
- тест `Solver.FindsTwoRoots` проверяет квадратное уравнение с двумя корнями;
- тест `Solver.RejectsLinearEquation` проверяет обработку неверного коэффициента `a = 0`; 
- тесты встроены в CMake и могут запускаться локально или в CI.

Проект получил автоматические проверки, которые подтверждают корректность основных библиотек после изменений.
